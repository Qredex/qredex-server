# Security Policy

## Reporting a Vulnerability

Do not open public GitHub issues for security vulnerabilities.

Report security issues privately to:

- `security@qredex.com`

Please include:

- affected package and version
- clear reproduction steps or proof of concept
- impact assessment if known
- any relevant logs, request IDs, or traces with secrets removed

Qredex will acknowledge receipt as quickly as possible and coordinate remediation privately until a fix is ready to disclose.

## Supported Versions

Security fixes are prioritized for the latest published version of `qredex`.

If a vulnerability affects older versions, Qredex may ask integrators to upgrade to a supported release rather than backporting fixes.
